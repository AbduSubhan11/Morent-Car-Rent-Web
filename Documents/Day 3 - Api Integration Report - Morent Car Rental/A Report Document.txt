--> API INTEGRATION PROCESS <--

Step 1: Fetch Car Data
The car data is retrieved from an external API endpoint:
https://sanity-nextjs-application.vercel.app/api/hackathon/template7
Each car object contains fields such as id, name, brand, type, fuel_capacity, transmission, seating_capacity, price_per_day, original_price, and image_url.

Step 2: Upload Car Images
The image_url field from the API is used to upload images to Sanity.
Using the Sanity client, the image is downloaded as a buffer using axios and then uploaded as an asset to Sanity.
Once uploaded, the image assetâ€™s _id is stored and referenced in the image field of the car object.

Step 3: Prepare Data for Sanity
After processing the image, a new car object is created with the required fields and mapped to the Sanity schema.
Optional fields (like original_price and tags) are set to null or default values if not present in the API response.

Step 4: Upload Car Data to Sanity
The transformed car object is uploaded to Sanity using the create method of the Sanity client.
Each car document is assigned a unique _id and includes all necessary data for rendering on the front end.
